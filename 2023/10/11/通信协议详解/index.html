IIC

​ I2C串行总线一般有两根信号线，一根是双向的数据线SDA，另一根是时钟线SCL。所有接到I2C总线设备上的串行数据SDA都接到总线的SDA上，各设备的时钟线SCL接到总线的SCL上。速率最高400Kbit/s。

​ IIC总线上可以挂很多设备：多个主设备，多个从设备（外围 设备）。

​ 多主机会产生总线裁决问题。当多个主机同时想占用总线时，企图启动总线传输数据，就叫做总线竞争。I2C通过总线仲裁，以决定哪台主机控制总线。 IIC和CAN一样，都有硬件的仲裁，但是软件IIC仲裁的话，可以在程序执行。

​ 起始条件：SCL线是高电平时，SDA线从高电平向低电平切换。

​ 停止条件：SCL线是高电平时，SDA线从低电平向高电平切换。

​ 每个接到I2C总线上的器件都有唯一的地址。主机与其它器件间的数据传输可以是由主机发送数据到其它器件，这时主机 即为发送器，总线上收数据的器件则为接收器。

IIC读取一字节数据流程

​ IIC_Start(); // 1起始信号
​ IIC_Send_Byte(WRITE_ADDRESS); // 2发送从设备地址
​ IIC_Wait_Ack(); // 3等待应答
​ IIC_Send_Byte(reg); // 4发送寄存器地址
​ IIC_Wait_Ack(); // 5等待应答
​ IIC_Stop(); // 6发送停止信号
​ IIC_Start(); // 7发送起始信号
​ IIC_Send_Byte(READ_ADDRESS); // 8发送读地址
​ IIC_Wait_Ack(); // 9等待应答
​ data=IIC_Read_Byte(); // 10接收数据
​ IIC_NAck(); // 11非应答接收 和 应答接收区别就是发不发应答信号。
​ IIC_Stop(); // 停止

CAN

CAN 总线电平为隐形时 电平为1， 为显性时 电平为0。 在CAN总线 显性电平优先级大于隐形。由于总线仲裁的原因。

CAN 错误帧：

一．错误帧由错误标志位和错误界定符两个部分组成。

  1. 如果在发送和接受报文时出现帧错误，那么该节点就会发送错误帧，提示该总线上其他节点自己出错了，之后总线会将该节点屏蔽。

  2. CAN错误标志分为三种：

​ 一般来说CAN是不允许超过5个相同信号的发送，但是错误帧除外。

​ a) 主动错误标志：6个连续的显性位，也就是如果出现主动错误，那么连续发送6个 低电平（0）。

​ b) 被动错误标志：6个连续的隐形位，也就是如果出现被动错误，那么连续发送6个 高电平（1）。

​ c) 错误界定符：8个连续的隐形位，也就是如果出现错误界定符，那么连续发送8个 高电平（1）。

CAN的错误帧一共有5种：位错误，ACK错误，填充错误，CRC错误，格式错误。

\1. 位错误：节点在向总线发送电平的同时，同时也读取总线上的电平，如果发现2者不一致，那么这个节点就检测出一个位错误，但是有三种情况不属于位错误：

a) 在仲裁区，节点向总线发送隐形位，但是在总线却读到了显性，不认为是位错误，这种情况对应仲裁失败，个人理解为：由于CAN总线的仲裁是在有2个节点或2个以上节点同时发送消息时，通过在仲裁段的第一位开始判断那个节点连续输出显性电平多，例如该总线当前有三个节点，分别标号为A、B、C，与此同时，节点A和节点B同时发送数据，节点A发送数据0x123455，节点B发送数据0x123456，将其转为二进制那么将其对应到二进制为：

节点A数据为：100100011010001010101

节点B数据为：100100011010001010110

那么开始进行判断从最高位为1001000110100010101 在这之前数据都是相同的，直到倒数第二位开始节点B的数据为1为隐形电平，节点A的数据位0为显性电平，由于仲裁规则为由仲裁段从第一位开始，哪一个节点连续发送的显性电平也就是低电平多，谁就仲裁获利，另一个仲裁失利。 所以可以判断出本次仲裁A节点胜出。 本次发送节点A的消息。

b) 在ACK槽，节点向总线发送隐形电平，却读到显性位，不认为是位错误，这种表示，该节点当前发送的这一帧报文至少被一个其他节点正确接收。

名词解释：

​ CAN分为几帧：

​ 数据帧，遥控帧，错误帧， 过载帧，帧间隔。

数据帧又分为：

IDE 标志拓展位，RTR位Tranmission Request Bit （发送请求位）。

RTR位Tranmission Request Bit （发送请求位）。RTR位的作用：在ID号相同的情况下，保证数据帧的优先级高于遥控帧。这么做的原因是保证数据帧优先级高于遥控帧。比如：在某一时刻t，节点Node_A发出了ID号为ID_2遥控帧报文来请求总线上的其它节点发出ID号为ID_2的数据帧报文。但是就在同一时刻t，节点Node_B发出了ID号为ID_2的数据帧报文。这个时候怎么办呢，显然依靠ID号不能仲裁出这两帧报文（一个遥控帧，一个数据帧，ID号相同）谁能占据总线的发送权，这种情况下，RTR位就起作用了，由于RTR在数据帧中恒为显性0，在遥控帧中恒为隐性1，所以在ID号相同的情况下，一定是数据帧仲裁获胜。这就解释了 RTR位的作用：在ID号相同的情况下，保证数据帧的优先级高于遥控帧。

Substitutes for Remote Requests Bit（替代远程请求位）：

Substitutes for Remote Requests Bit（替代远程请求位），在扩展帧（数据帧或遥控帧）中，SRR恒为隐性位1，并且可以发现，扩展帧的隐性SRR位正好对应标准帧的显性RTR位，这就解释了 SRR位的作用：在前11位ID号相同的情况下，标准数据帧的优先级高于扩展数据帧。

线与：

总线上的电平有显性电平和隐性电平两种。
总线上执行逻辑上的线“与”时，显性电平的逻辑值为“0”，隐性电平为“1”。
“显性”具有“优先”的意味，只要有一个单元输出显性电平，总线上即为显性电平。并且，“隐
性”具有“包容”的意味，只有所有的单元都输出隐性电平，总线上才为隐性电平。（显性电平比
隐性电平更强。）

标准帧：

​ 帧起始（SOF）：表示帧开始的段。 1 个位的显性位

​ 仲裁段（ID）：表示数据的优先级的段，标准格式和扩展格式在此的构成有所不同 。

​ 标准格式的 ID 有 11 个位。从 ID28 到 ID18 被依次发送。禁止高 7 位都为隐性。（禁止设定： ID=1111111XXXX）扩展格式的 ID 有 29 个位。基本 ID 从 ID28 到 ID18，扩展 ID 由 ID17 到 ID0 表示。基本 ID 和标准格式的 ID 相同。禁止高 7 位都为隐性。（禁止设定：基本 ID=1111111XXXX）。

​ 控制段（）：控制段由 6 个位构成，表示数据段的字节数。标准格式和扩展格式的构成有所不同。

​ 1。保留位（ r0、 r1）

​ 保留位必须全部以显性电平发送。但接收方可以接收显性、隐性及其任意组合的电平。
